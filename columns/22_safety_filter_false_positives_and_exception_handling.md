# セーフティフィルターの「誤検知（偽陽性）」とポリシー例外によるAIシステムの即死

## 1. 概要
LLM（大規模言語モデル）の商用API（Azure OpenAI や Google Vertex AI、Anthropic API 等）を利用する際、システム設計上の死角となりやすいのが「セーフティフィルター（コンテンツモデレーションポリシー）」の突発的な発動である。
Onyx（`onyx-dot-app/onyx#10893`）および Claude Code（`anthropics/claude-code#71883`）のインシデント事例は、プロバイダ側の安全フィルターによる誤検知（偽陽性）が発生した際、呼び出し側システムが例外処理（Exception Handling）を怠っていると、プロセス全体がクラッシュして即死するか、エージェントが開発デッドロックに陥るリスクを示している。
本稿では、セーフティブロックがもたらすシステム破壊のメカニズムと、その防衛設計について考察する。

---

## 2. インシデントの分析：セーフティブロックの暴発とハンドリング漏れ

### 2.1. 未ハンドリング例外によるプロセスの突然死
RAGシステムやAIチャットにおいて、取得したドキュメントやユーザーの入力、あるいはモデル自身の回答が Azure OpenAI などのポリシーフィルターに引っかかった際、API はHTTP 400（BadRequest）とともにポリシー違反を示す特殊なエラーを返却する（`onyx#10893`）。

中継ライブラリ（LiteLLMなど）はこれを **`ContentPolicyViolationError`** などの固有の例外として送出するが、アプリケーションがこの例外を明示的にキャッチしていないと、以下の問題が発生する。

* チャットセッションやバックグラウンドのドキュメントインデックス処理を担当するバックグラウンドワーカーが、unhandled exception（未処理の例外）によって**プロセスごと突然死（異常終了）**する。
* システム全体がフリーズするか再起動ループに陥り、他の無関係なユーザーのサービスまで巻き添えで停止する。

### 2.2. 正当な開発コードの誤検知（偽陽性）による「検閲デッドロック」
AIエージェントにネットワーク中継（例：RTSPストリームのプロキシ）や、セキュリティ診断スクリプト、システムユーティリティなどの低レイヤーなコードを記述させようとする際、プロバイダのサイバーセキュリティフィルター（Cybersecurity filter）が暴発することがある（`claude-code#71883`）。

* **エラー**: `API Error: Safety filter blocked...`
* プロバイダは、悪意のない正当な開発コードを「マルウェア」や「ハッキング行為」と誤認して出力を遮断する。
* これにより、AIエージェントは「コードをコピーしようとするとAPIから強制的に出力を止められる」状態になり、パースエラーを起こして死ぬか、あるいは同じ処理を無限に再試行してトークンを虚無に溶かし続ける（検閲デッドロック）。

---

## 3. セーフティブロックに対する「フォールバック設計」

AIエージェントやLLM駆動システムを本番稼働させるためには、プロバイダ側のコンテンツフィルタは「いつでも突発的にエラーを返してくるもの」という前提（Design for Failure）でインフラを設計しなければならない。

### 3.1. ポリシー違反例外の「全件キャッチ」と安全なフォールバック
* `ContentPolicyViolationError` などのプロバイダ固有のポリシー例外を、例外処理の最上位レイヤーで明示的にトラップ（キャッチ）する。
* プロセスを即死させるのではなく、AIエージェントに対して「APIポリシー違反により出力を継続できませんでした。入力やコード内のセンシティブと思われる表現（あるいは脆弱性コードの例）を修正または回避してください」とエラーを適切にフィードバックし、安全にタスクをリカバリ（または一時停止）させる。

### 3.2. プロバイダの「冗長化（マルチLLMフォールバック）」
* 特定のプロバイダ（例：Azure OpenAI）のセーフティフィルターが過敏に反応してブロックされた場合、自動的に別の安全基準を持つプロバイダ（例：Anthropic または ローカルのプライベートLLM）に一時的にリクエストを切り替える（フォールバック）ルートをゲートウェイ（LiteLLM等のプロキシ層）に実装する。
* これにより、フィルターの誤検知によって開発プロセス全体が完全に停止するリスクを回避する。

---

## 4. 結論
セーフティフィルターによる出力ブロックは、AIエージェント開発における「防ぎようのない外部要因」である。
これに対し、
1. **ポリシー例外を確実にトラップしてプロセスの突然死を防ぐ堅牢な例外ハンドリング**
2. **誤検知発生時に別モデルへ処理を逃がす冗長化（フォールバック）設計**

をシステム層に備えることこそが、プロバイダの気まぐれな「検閲」に振り回されず、24時間365日安定して動作する自律型AIインフラを構築するための不可欠なセキュリティ要件である。
